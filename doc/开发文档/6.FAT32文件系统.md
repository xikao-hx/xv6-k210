# fat32

**代码流程总结**：
* dirlookup/ename 负责通过目录项找到 struct dirent；
* entry->first_clus 是文件数据的入口；reloc_clus() 根据 offset 沿 FAT 表调用 read_fat() 找到当前簇；
* 最后 rw_clus() 用 first_sec_of_clus() 把簇号转成扇区号，并通过 bread/bwrite 完成实际 I/O。

## fat32 基本知识
### 磁盘布局
+--------------------+
| Boot Sector / BPB  |
+--------------------+
| Reserved Sectors   |
+--------------------+
| FAT Table 1        |
+--------------------+
| FAT Table 2        |
+--------------------+
| Data Region        |
|  - directories     |
|  - file contents   |
+--------------------+

**其中最重要的是**：
* Boot Sector / BPB：记录文件系统参数，比如每扇区多少字节、每簇几个扇区、FAT 表多大、根目录在哪个簇。
* FAT Table：文件分配表，记录每个簇的下一个簇号。
* Data Region：真正存放文件内容和目录内容的区域。

### FAT 表

**总结概括**：
* 每个 FAT32 表项占 4 字节，FAT 表中每个 entry 记录“当前簇的下一个簇号”。文件内容不是连续块，而是一条簇链。
* FAT32 的数据簇编号从 2 开始

**FAT 表的作用**：
* FAT 表可以理解成一个数组：
```text
FAT[cluster_number] = next_cluster_number
```

* 特殊值：
```text
0        表示空闲簇
EOC      表示簇链结束
bad      表示坏簇
其他值   表示下一个簇号
```

* 所以读文件时，逻辑是：
```text
从目录项拿到 first_clus
读这个簇的数据
查 FAT[first_clus] 得到下一个簇
继续读
直到遇到 EOC
```

* 写文件时，如果文件变大，需要：
```text
找一个空闲簇
把当前最后一个簇的 FAT 项指向新簇
把新簇 FAT 项设为 EOC
```

**如何通过 FAT 表记录一个文件**：
```text
文件 A:
cluster 5 -> cluster 8 -> cluster 20 -> EOC

FAT 表里大概是：
FAT[5]  = 8
FAT[8]  = 20
FAT[20] = EOC
EOC 表示 End Of Chain，也就是文件结束。
```

### 目录项

FAT32 目录内容是一堆 32 字节 entry

**短目录项/长目录项**：
* 短目录项才真正保存：
```text
属性
起始簇
文件大小
```
* 长目录项主要保存文件名片段。

**三类情况**：
```text
0x00 END_OF_ENTRY   后面没有目录项了
0xe5 EMPTY_ENTRY    这个目录项被删除或空闲
0x0f ATTR_LONG_NAME 长文件名项
普通 short entry    真正保存文件元数据
```

## fat32 目录查找

**union dentry 和 struct dirent 的关系**：
* union dentry 是 FAT32 磁盘上的 32 字节原始目录项格式；
* struct dirent 是内核为了操作文件方便，抽象出来的内存文件对象。

**目录的查找过程**：
```text
1. path 以 / 开头，从 root 开始
2. skipelem 得到 "home"
3. dirlookup(root, "home")
   - root 是目录文件
   - enext 扫描 root 的目录项
   - 找到 home 的目录项
   - 得到 home.first_clus

4. skipelem 得到 "user"
5. dirlookup(home, "user")
   - 根据 home.first_clus 读取 home 目录内容
   - 如果目录跨簇，通过 FAT 表继续找
   - 找到 user 的目录项

6. skipelem 得到 "a.txt"
7. dirlookup(user, "a.txt")
   - 扫描 user 目录文件
   - 解析长文件名项和短目录项
   - 找到 a.txt
   - 得到 a.txt.first_clus 和 file_size

8. 返回 a.txt 对应的 struct dirent
```

**簇的使用过程**：
```text
root dirent
  first_clus = root_clus
      |
      v
扫描 root 目录文件
      |
      v
找到 "home" dirent
  first_clus = 10
      |
      v
扫描 home 目录文件
      |
      v
找到 "user" dirent
  first_clus = 25
      |
      v
扫描 user 目录文件
      |
      v
找到 "a.txt" dirent
  first_clus = 40
  file_size = 1234
```

## fat32 读写

**读文件**：
```text
read(fd)
  -> fileread()
    -> eread()
      -> reloc_clus()
        -> read_fat()
      -> rw_clus()
        -> bread()
        -> copyout()
```

**写文件**：
```text
write(fd)
  -> filewrite()
    -> ewrite()
      -> 如果空文件，alloc_clus()
      -> reloc_clus(..., alloc=1)
        -> read_fat()
        -> 必要时 alloc_clus()
        -> write_fat()
      -> rw_clus()
        -> bread()
        -> copyin()
        -> bwrite()
      -> 更新 file_size
      -> 标记 dirty
```

## 代码总结

**总结**：
* 就是通过目录查找得到文件的目录项，然后得到第一个簇号，再通过 FAT表的簇链，来查找 offset 对应的簇号，来完成读写这文件
* 然后读写基本单位可以看作是读写簇，但是其实也就是在读写扇区

**先看 FAT32 初始化**：fat32_init
* 重点理解它从 0 号扇区读 BPB：
* 每扇区字节数 byts_per_sec
* 每簇扇区数 sec_per_clus
* FAT 表数量 fat_cnt
* FAT 表大小 fat_sz
* 根目录起始簇 root_clus
* 数据区起始扇区 first_data_sec

**再看“簇号怎么变成磁盘位置”**：
* first_sec_of_clus()：簇号 -> 数据区扇区号
* fat_sec_of_clus()：某个簇在 FAT 表的哪个扇区
* fat_offset_of_clus()：某个簇在 FAT 扇区内的偏移

**然后看 FAT 表读写和簇分配**：
* read_fat(cluster)：读当前簇的下一个簇
* write_fat(cluster, content)：修改 FAT 表
* alloc_clus()：扫描 FAT 表，找值为 0 的空闲簇
* free_clus()：把 FAT entry 清 0
* zero_clus()：新分配的簇清零

**再看文件读写**：
* reloc_clus(entry, off, alloc)：根据文件偏移找到对应簇
* eread()：按簇读取
* ewrite()：按簇写入，必要时扩展簇链
* rw_clus()：真正调用 buffer cache 读写扇区

**再看目录项和长文件名**：
* FAT32 目录本质上也是一个文件，里面每 32 字节一个目录项。
    * 短目录项：保存属性、起始簇、文件大小
    * 长目录项：保存长文件名片段
* emake()：把内存中的 dirent 写成磁盘目录项
* enext()：遍历目录，解析长目录项 + 短目录项
* dirlookup()：在目录中查找文件名

**最后看路径解析和系统调用接入**：
* ename(path) 类似 xv6 原来的 namei
* enameparent(path, name) 类似 xv6 原来的 nameiparent
* sys_open() 调 ename/ealloc
* fileread() 调 eread
* filewrite() 调 ewrite
