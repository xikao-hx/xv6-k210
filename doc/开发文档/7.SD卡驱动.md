# SD卡驱动

**代码流程总结**：
* 初始化阶段：按照 SD 规范流程，先发送 80 个空时钟，再用 CMD0 进入 SPI 模式，用 CMD8 和 CMD58 检查电压和 OCR，用 CMD55+ACMD41 完成初始化，最后根据 OCR 的 CCS 位判断是 SDSC 还是 SDHC/SDXC。这个判断会影响后续读写寻址：SDSC 使用字节地址，SDHC/SDXC 使用 512 字节块号。
* 读写层面：实现了单块读写。读扇区使用 CMD17，等待 R1 成功响应和 0xfe start token 后，用 DMA 读 512 字节；写扇区使用 CMD24，发送 0xfe start token、512 字节数据和 dummy CRC，然后检查 data response token，并用 CMD13 确认写入状态。
* 分层：上层文件系统不直接接触 SD 协议，而是通过 buffer cache 的 bread/bwrite 访问扇区，最终由 disk.c 调到 SD 卡驱动。

## SD卡初始化流程

**初始化流程：**
```text
1. 发送至少 80 个 SPI clock，让 SD 卡进入可接收命令状态
2. CMD0：进入 SPI 模式，并进入 idle 状态
3. CMD8：检查电压范围和版本
4. CMD58：读取 OCR，确认电压范围
5. CMD55 + ACMD41：让 SD 卡完成初始化，退出 idle
6. CMD58：再次读取 OCR，判断 SDSC 还是 SDHC/SDXC
7. 如果是 SDSC，用 CMD16 设置 block size
```

**代码对应：**
```text
switch_to_SPI_mode();          // CMD0
verify_operation_condition();  // CMD8
read_OCR();                    // CMD58
set_SDXC_capacity();           // CMD55 + ACMD41
check_block_size();            // CMD58 + CMD16
```

**SDSC 和 SDHC/SDXC 的寻址方式不同：**
```text
SDSC:
  CMD17/CMD24 参数是字节地址
  所以 sectorno * 512，也就是 sectorno << 9

SDHC/SDXC:
  CMD17/CMD24 参数是块号
  所以直接传 sectorno
```

## 读扇区流程

**读一个 512 字节扇区的流程：**
```text
1. 根据卡类型计算 address
2. 加 sdcard_lock，保证一次 SD 操作不会被并发打断
3. 发送 CMD17，参数是 address
4. 读取 R1 response，确认命令成功
5. 等待 data start token 0xfe
6. 通过 DMA 读取 512 字节数据
7. 读取并丢弃 2 字节 CRC
8. 拉高 CS，结束命令
9. 释放锁
```

**核心代码逻辑：**
```c
sd_send_cmd(SD_CMD17, address, 0);
result = sd_get_response_R1();

while (...) {
  sd_read_data(&result, 1);
  if (0xfe == result) break;
}

sd_read_data_dma(buf, BSIZE);
sd_read_data(dummy_crc, 2);
sd_end_cmd();

// SD 卡读数据不是 CMD17 后立刻返回数据，中间会有若干无效字节，直到出现 0xfe start token。
```

## 写扇区流程

**写一个 512 字节扇区的流程：**
```text
1. 根据卡类型计算 address
2. 加 sdcard_lock
3. 发送 CMD24，参数是 address
4. 读取 R1 response，确认命令成功
5. 发送 start block token 0xfe
6. 通过 DMA 写入 512 字节数据
7. 发送 2 字节 dummy CRC
8. 等待 Data Response Token，确认 SD 卡接收成功
9. 等待 SD 卡内部编程完成
10. 发送 CMD13 检查写入状态
11. 结束命令，释放锁
```

**代码里的关键点：**
```c
sd_send_cmd(SD_CMD24, address, 0);
sd_write_data(&START_BLOCK_TOKEN, 1);
sd_write_data_dma(buf, BSIZE);
sd_write_data(dummy_crc, 2);

// 写完后它还会检查：
if (0x05 == (result & 0x1f))

// 0x05 表示 data accepted。然后再用 CMD13 查状态，确认写入没有错误。
```
